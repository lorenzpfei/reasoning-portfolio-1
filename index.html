Experiment: The Cost of Patience

So far, \(\gamma = 1.0\): a win in 10 flips is worth exactly the same as a win right now. But what if future rewards are worth a little less than immediate ones? This is the discount factor \(\gamma\). Drop it below 1 and the agent starts to care about how long it takes to win – not just whether it wins.

Why this matters

In most real RL problems, \(\gamma < 1\) is used for two reasons: it keeps the math well-behaved (prevents infinite sums), and it models the fact that getting something sooner is usually better than getting it later. In the Gambler’s Problem the book uses \(\gamma = 1\) because the game always ends – there is no infinite loop to worry about.

Developer Insights

The following two issues are not in the book. They come up the first time you implement this yourself.

Issue #1: Float Comparison

Problem

// Naive convergence check
if (newValue !== oldValue) keepIterating();

Why does this happen?

0.4 × 1.0 in IEEE 754 = 0.40000000000000002

Strict equality never returns true for most floating point results.

The values computed are correct – only the termination condition is broken.

Fix — epsilon tolerance \(\theta = 10^{-9}\)

// With epsilon tolerance
if (Math.abs(newValue - oldValue) > 1e-9) keepIterating();

Issue #2: Tie-Breaker

Problem

// Naive argmax – takes last maximum found (= largest bet)
let bestAction = 1, bestValue = -Infinity;
for (const a of actions) {
  if (bellman(s, a) >= bestValue) {
    bestAction = a;
    bestValue = bellman(s, a);
  }
}

Why does this happen?

At many states, multiple bet sizes produce exactly the same \(V(s)\).

For example at s=50: bets of $50, $25, $12… can all be equally optimal.

argmax with no tie-breaking depends on iteration order, not mathematics.

Sutton & Barto call this “a whole family of optimal policies” – your code just happens to pick one member arbitrarily.

Fix

// On tie: prefer smallest bet
if (qValue > bestValue ||
   (qValue === bestValue && a < bestAction)) {
  bestAction = a;
  bestValue = qValue;
}
